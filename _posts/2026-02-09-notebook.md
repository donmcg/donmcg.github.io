Query: "update jekyll to support gfm addons"

AI says:

To update Jekyll and enable GitHub Flavored Markdown (GFM) add-ons, update your Gemfile to use github-pages or newer kramdown-parser-gfm, then run bundle update. Configure _config.yml to use kramdown and ensure GFM plugins (like jekyll-gfm-admonitions) are added to the :jekyll_plugins group in your Gemfile. 

**Steps to Update and Enable GFM Add-ons:**

- **Update Gemfile:** Ensure your Gemfile uses the github-pages gem to match GitHub's environment, which supports GFM.


{% highlight ruby %}
ruby

gem "github-pages", group: :jekyll_plugins

{% endhighlight %}

- **Add GFM Plugins:** Add desired plugins to the :jekyll_plugins group,
such as gem "jekyll-gfm-admonitions".
- **Update Dependencies:** Run bundle update in your terminal to
install the latest versions of the gems.
- **Configure _config.yml:** Set kramdown as the markdown
engine and enable GFM support.

{% highlight yaml %}
yaml

markdown: kramdown
kramdown:
  input: GFM
plugins:
  - jekyll-gfm-admonitions
 
{% endhighlight %}

For troubleshooting, delete Gemfile.lock and run bundle install again if errors persist.

What I did:

- updated _config.yml:

```yaml

# experimental use of kramdown
markdown: kramdown

# attempting to enable Alerts 9feb2026 
kramdown:
  input: GFM
plugins:
  - jekyll-gfm-admonitions
```

- updated Gemfile:

```ruby

# If you want to use GitHub Pages, remove the "gem "jekyll"" above and
# uncomment the line below. To upgrade, run `bundle update github-pages`.
gem "github-pages", group: :jekyll_plugins

# If you have any plugins, put them here!
group :jekyll_plugins do
 gem "jekyll-gfm-admonitions", "~> 0.2"
end

```

- issued bash commands

```bash

$ # stop the serve script

$ cd ~/mai/projects/github/donmcg.github.io/

$ vi _config.yml # edit as shown above 
$ vi Gemfile     # edit as shown above;
                 # makes jelyll-gfm-admonitions visible to bundle

$ brew update
$ brew upgrade

$ bundle install    # this should add the jekyll-gfm-admonitions plugin
  # once the above command executes the new gem should appear in Gemfile.lock

$ grep gfm Gemfile.lock  # check to see if it's there?   Yes!
      kramdown-parser-gfm (= 1.1.0)
    jekyll-gfm-admonitions (0.2.1)
    kramdown-parser-gfm (1.1.0)
  jekyll-gfm-admonitions (~> 0.1) 

$ # don't do this # $ bundle update github-pages
  # this is for coordinating multiple sites

$ serve
  # open this page and see that alerts below are working

```
